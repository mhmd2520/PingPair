fping 5.5 — flag reference

PingPair ships fping 5.5 (Cygwin build) under bin\fping\. The flags below are the ones PingPair sets when running a case.

Command PingPair builds (per case)

fping <target> -S <source> -p 10 -c <count> -s -D

Example for case #01 payload=200B bw=10Mbps on a real LAN:

fping 192.168.1.1 -S 192.168.1.2 -p 10 -c 3000 -s -D

Flags used

FlagMeaningWhy PingPair uses it
-c <count> Send exactly count ICMP echo requests, then exit. Lets fping self-terminate at the end of a case so it can flush its summary block. PingPair sets count = duration_s × 1000 ÷ interval_ms.
-p <ms> Interval between consecutive pings to the same target, in milliseconds. 10 ms (the procedure value). Note: Windows' default timer granularity is ~15.6 ms, so the actual interval is ~16 ms.
-l "Loop forever" mode — without -c this never exits on its own. PingPair only uses this in the preview CLI string (to match the written procedure verbatim); at runtime it substitutes -c <count> for clean termination.
-s Print summary statistics (totals, min/avg/max RTT, elapsed time) when fping ends. This is what PingPair's report parser reads to fill in the Min/Avg/Max latency columns.
-D Prefix each line with a high-precision Unix timestamp. Used by the live-latency chart in the Run tab.
-S <ip> Set the source address for outbound packets. In two-machine mode, pins the Client side's source to 192.168.1.2. Skipped in loopback mode (both sides are 127.0.0.1).

Exit / signal behaviour

With -s -c <count> fping exits cleanly with rc=0 and prints its summary to stderr (not stdout — that's an fping ≥ 4.0 change). PingPair's parser combines both streams before extracting metrics, so the summary always lands in the report.
