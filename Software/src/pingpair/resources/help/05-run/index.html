Running a sweep

The Run tab is where the test actually happens. What it shows depends on this PC's role — the Client drives, the Server obeys, and Loopback does both on one machine.

The Run tab as it looks for this PC's current role — the case table on the left, live throughput + latency charts on the right. Click the image to enlarge.

How the two sides talk

The Client drives; the Server obeys. They coordinate over a control channel on TCP 5202 — separate from iperf3's own data port (5201).

The control-channel handshake: a HELLO/READY greeting, a per-case START/READY/DONE/RESULT exchange for all 20 cases, then FINISH — both ends save.

On the Server

The Server side mostly waits. It listens on TCP 5202 for the Client's control connection and starts/stops iperf3 for each case on command. The panel shows a status line — "Listening on 192.168.1.1…" — and a live log.

"Waiting for … to come up" is normal

If the Server's IP isn't bound yet (e.g. you just booted on DHCP and haven't run Set the correct IP), the listener says it's waiting and keeps retrying. The moment the IP is set, it starts listening on its own — no app restart needed.

On the Client

Two ways to run:

  • Run full sweep — runs all cases in the plan.
  • Run subset (N cases) — tick individual rows in the table first, then run only those. Useful for re-testing a single payload/bandwidth combination.

As the sweep runs, the case table fills in row by row, a progress bar tracks completion, and a whole-sweep ETA / elapsed time readout adapts as cases finish. The charts on the right show live per-second throughput and latency for the case in flight.

Expect ~50 s per case, not 30 s

Each case is a 30 s iperf3 run plus an fping run. Windows' timer granularity (~15.6 ms) stretches fping's 10 ms ping interval, so a full 20-case sweep takes roughly 16 minutes. This is expected, not a hang.

Stopping early

The Run button becomes a Stop while a sweep is in flight. Stopping (or a connection error) ends the sweep cleanly and re-enables the Run button with the right label — completed cases keep their results.

When it finishes

A popup summarises the run: how many cases succeeded out of the total, the total duration, and — via its Details button — every file that was saved. What gets saved (and whether you're prompted first) is controlled on the Save Options tab.

Before your first real run

Use the Ping tab to smoke-test reachability: target the other side's IP and press Ping. A clean, 0% loss result means the LAN is sane and the firewall isn't blocking ICMP. Fix the network first if this fails.
