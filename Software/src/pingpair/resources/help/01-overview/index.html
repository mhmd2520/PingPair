Overview — the architecture in diagrams

A diagram-led tour of how PingPair is put together, ordered the way the system actually flows: the physical test setup, the one-PC alternative, how the tabs fit together, the four-step operating path, what happens on the wire per case, the 20-case matrix, and finally what a run leaves on disk. New here? Take the Quick Start first — this page is the bigger picture behind it.

1 · Test topology

Where it all starts: two laptops wired back-to-back over a single Ethernet cable — no switch or router. The Server listens on 192.168.1.1; the Client drives the sweep from 192.168.1.2.

The standard two-laptop test topology.

2 · Loopback (one-PC) topology

No second machine? Loopback collapses both roles onto one PC over 127.0.0.1 — the same 20-case grid and the same report, with no cable. Ideal for a dry run before you wire two laptops together.

Loopback mode — both roles on a single PC.

3 · How the tabs fit together

The application is organised as a left-to-right flow: prepare the link (Setup, Ping, Config), run the sweep (Run), then read the results (Save Options, Analysis) — with Help backing every step.

Prepare → run → read the results.

4 · The four-step operating path

The day-to-day flow once the app is open: wire & launch, get every Setup check green with Fix all, run the full sweep, then collect the report set when the finish popup appears.

Wire & launch → Setup (Fix all) → Run → collect the report.

5 · The control sequence

How the two PCs stay in step. They talk over a private control channel on TCP 5202 (separate from the iperf3 data port 5201). For each case the Client tells the Server to start, both ends measure, and the Client collects the result before advancing — neither side moves on until the case is done.

The per-case handshake — the Client orchestrates, the Server obeys.

6 · The 20-case grid

What actually gets measured: four payload sizes × five offered bandwidths = 20 cases, each run for ~30 seconds. Tune the grid on the Config tab.

Four payloads × five bandwidths = the 20 cases.

7 · What a run produces

The output of every sweep: its own folder holding a Word .docx, an Excel .xlsx, and a .json sidecar by default — plus .pdf / .txt on request. Set it all up on Save Options.

One self-contained folder per run.
