Ping — reachability check

The Ping tab is a quick smoke test you run before a full sweep. It wraps Windows' built-in ping — no Cygwin involved — to confirm the two laptops can actually see each other over the Ethernet link. If this fails, fix the network or the firewall before you bother with a 16-minute sweep.

The target defaults to the other PC

PingPair pre-fills the target with whichever end you'd be testing against:

This PC's roleDefault target
Serverthe Client (192.168.1.2)
Clientthe Server it sweeps (192.168.1.1, or your host override)
Loopback127.0.0.1

Change the role and the default follows — but anything you typed yourself is preserved.

Reading the summary

When the run finishes, the summary panel parses the key numbers out of ping's output so you don't have to:

  • Packets sent / received and lost — the raw counts.
  • Loss — the percentage. On a healthy direct link this should be 0%.
  • Min / Avg / Max RTT — round-trip time in milliseconds. On a back-to-back Ethernet link expect well under 1 ms.

Ping passing isn't the whole story

A clean ping proves ICMP gets through, but the sweep also needs the iperf3 data port and the control channel open. If ping is perfect yet a sweep still can't connect, it's a firewall port, not the cable — see Troubleshooting.

Seeing phantom loss?

Non-zero loss on a direct cable usually means a Wi-Fi adapter is stealing the 192.168.1.x traffic. Disable it from the Setup tab and ping again.
