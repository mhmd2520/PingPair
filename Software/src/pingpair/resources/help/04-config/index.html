Config — the test plan

The Config tab defines what gets tested: the payload sizes, the bandwidths, and the network addresses each side uses. The defaults already match the standard procedure, so you can usually leave this alone.

The Config tab — the Test plan box (payloads, bandwidths, duration, protocol, IPs, ports), the live Raw JSON pane, and the Live CLI preview of the exact commands a sweep will run. Click the image to enlarge.

The 20-case grid

A sweep is every payload crossed with every bandwidth. The defaults are four payloads × five bandwidths = 20 cases, each running ~30 s:

Payload (bytes)200, 600, 1000, 1300
Bandwidth (Mbps)10, 30, 50, 70, 90

The 20 cases — four payloads down, five bandwidths across, run in order 1 → 20.

Edit either list in the Test plan box as comma-separated numbers. The case count updates to match (e.g. 3 payloads × 4 bandwidths = 12 cases).

Network addresses

The canonical point-to-point setup binds the two laptops back-to-back over Ethernet:

FieldDefaultMeaning
Server IP192.168.1.1The side that listens.
Client IP192.168.1.2The side that drives the sweep.
Subnet mask255.255.255.0/24 — both ends on the same subnet.
Gateway(blank)Leave empty for a point-to-point LAN. Set it only if your test path crosses a router.

These IPs drive the checks too

Changing the Server/Client IPs here re-evaluates the Setup tab's role and NIC checks against the new values, and the Set the correct IP fix will assign whatever you set here.

Saving & loading plans

  • Apply to current session — make your edits live for this run (also rebuilds the Run tab's grid).
  • Save As… — write the current plan to a .json profile you can reuse.
  • Import config… — load a plan from a .json file.
  • Download Template… — save a blank/example profile to start from.
  • Reset to defaults — restore the standard 4×5 grid and IPs.

Live CLI preview

The Live CLI preview box shows the exact fping and iperf3 command lines your current plan will produce, so you can see precisely what each case runs before pressing Run. The Raw JSON box exposes the same plan as the JSON that gets stored alongside each report. For what every flag means, see the fping reference and iperf3 reference sections.
