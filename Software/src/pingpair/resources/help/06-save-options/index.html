Save Options & reports

The Save Options tab controls how each sweep's report set is written: the formats, where they land, optional record metadata, and whether PingPair asks you first.

The Save Options tab — the Save settings box (auto-save toggle, destination, filename pattern), the report-format checkboxes, and the optional Test-record metadata fields. Click the image to enlarge.

What a sweep writes

Each run gets its own folder under Reports\PingPair_<timestamp>\. The Word, Excel and JSON sidecar are always written; PDF and TXT are added when you tick them below.

One sweep → one folder: .docx + .xlsx + .json always, plus .pdf / .txt if ticked.

Prompt-first vs. auto-save

By default Auto save is off — when a sweep finishes, PingPair prompts you for a destination and filename. This keeps things explicit. Tick Auto save to instead write every run automatically to the configured Destination folder using the Filename pattern, hands-free.

What gets written

Each sweep produces a folder of artefacts:

FormatBy defaultNotes
Word (.docx)onThe formatted report matching the standard results table.
Excel (.xlsx)onThe same data as a spreadsheet, plus a Run-info sheet.
PDF (.pdf)offTick to also render a PDF.
Plain text (.txt)offTick for a no-frills text summary.
JSON sidecar (.json)alwaysThe exact config that produced the run — used by the Analysis tab and to reproduce a sweep.

Everything for one run lands together in a per-sweep subfolder named PingPair_<timestamp>, so runs never get mixed up. You can also embed the per-metric charts in the Word/PDF via the charts option.

Test-record metadata (optional)

Fill in fields like technician, customer, hardware serial, environment, and a record ID. Anything you enter is saved and embedded in the report headers; blank fields are simply omitted. These persist between runs.

Managing reports

  • Save report now — manually save the last sweep (handy when Auto save is off and you dismissed the prompt).
  • Open destination folder — jump to where reports are written.
  • Recent reports — lists report folders found in the destination, so previous-session runs survive a restart.
  • Reset to defaults — restore Auto-save off, the default destination and filename pattern (leaves metadata and the recent list alone).

Older / one-off files

The recent list only scans the per-sweep folder layout. To view a sidecar from somewhere else on disk, use the Analysis tab's Add file… button.
