Setup & prerequisites

The Setup tab is the first thing you see, because it gates everything else: it picks this PC's role, then runs a battery of checks that confirm the machine can actually run a sweep. Green across the board means you're ready to test.

The Setup tab in Server role — the role selector up top, then the ten prerequisite checks (here all green: "9 pass, 1 skipped", the skip being the optional gateway). Click the image to enlarge.

1. Which role is this PC?

PingPair runs on both laptops. One plays Server, the other plays Client — the Client drives the 20-case schedule and the Server obeys it. You don't pick the role from a dialog: on first launch PingPair reads this PC's bound IPv4 addresses and decides automatically:

This PC is bound to…Role chosen
192.168.1.1Server
192.168.1.2Client
neitherClient (safe fallback) + a one-time notice

A coloured banner across the top of the window always shows the current role. To change it, use the Role selector at the top of the Setup tab — Server, Client, or Loopback (a single-machine dev/smoke mode that runs both ends on 127.0.0.1). The Run tab rebuilds itself instantly; no restart needed. You can switch only while idle — switching mid-sweep is blocked.

Role / IP mismatch

If the saved role's expected IP isn't bound any more — say the NIC was reset or the laptop moved to another LAN — an orange banner appears. PingPair never silently flips your role; it asks you to either fix the IP or change the role. See Troubleshooting for the exact steps.

2. The prerequisite checks

PingPair runs ten checks, in this order. Each is colour-coded: green = good, yellow = a warning you can usually one-click fix, red = a blocker that will stop a sweep.

#CheckWhat it confirms
1Python interpreterPython 3.11+ is running the app.
2Administrator privilegesThe app is elevated, so the firewall / IP fixes can run. On Windows PingPair tries to auto-elevate at launch.
3fping binaryThe bundled fping.exe is present and responds.
4iperf3 binaryThe bundled iperf3.exe is present and responds.
5Local NIC IPThis PC holds the canonical IP for its role (Server 192.168.1.1 / Client 192.168.1.2).
6Firewall: ICMP echo (ping)An inbound rule lets ping through (fping's latency probes).
7Firewall: iperf3 (TCP/UDP 5201)Inbound rules allow the iperf3 data port.
8Firewall: control channel (TCP 5202)An inbound rule allows the Server↔Client control link.
9Gateway reachableOptional — only when a gateway is configured; pings it once. A point-to-point LAN has none, so this skips.
10Wi-Fi disabledWarns if a Wi-Fi adapter is carrying IPv4 traffic that could steal test packets from the Ethernet link.

3. Fixing what's not green

Most yellow/red rows carry a fix button that runs the exact command for you (a confirmation dialog shows the command first):

A red Local NIC IP row with its "Set the correct IP" action on the right, plus the orange role/IP-mismatch banner across the top (its "How to fix" button jumps to Troubleshooting). Click the image to enlarge.

  • Set the correct IP — assigns this PC's role IP to the primary Ethernet adapter via netsh.
  • Add ICMP rule — adds the inbound ping firewall rule.
  • Open 5201 (TCP+UDP) — adds the iperf3 data-port rules.
  • Open 5202 (TCP) — adds the control-channel rule.
  • Disconnect Wi-Fi — takes the Wi-Fi adapter offline for the test.
  • Restart as Administrator — relaunches elevated so the other fixes become available.

Two shortcuts

Fix all runs every available fix in sequence and shows a per-step transcript. Re-check re-runs all ten checks — handy after fixing something outside the app.

Wi-Fi is always restored

If PingPair disables Wi-Fi for a test, it re-enables it automatically when you close the app or click Reset all settings, and Windows reconnects to your saved network. Even a crash is covered — the next launch puts it back.

4. Starting over

Reset all settings (bottom of the Setup tab) is a factory wipe: it removes every PingPair firewall rule, reverts the primary Ethernet adapter to DHCP, re-enables Wi-Fi, clears all saved settings, and closes the app — leaving the PC the way a fresh Windows + fresh PingPair install would. Use it when you're done testing on a machine.
